# Tiger网球俱乐部 H5应用 PRD v1.0

> 产品需求文档。本文档锁定第一期功能范围，作为开发和设计的唯一依据。

---

## 一、产品概述

### 1.1 产品目标

为 Tiger 网球俱乐部提供三端一体的 H5 Web 应用：
- 学员/家长端：展示机构信息、约课、赛事报名、个人段位管理
- 教练端：课表管理、学员评估、月度反馈
- 管理员端：机构全局管理

### 1.2 用户角色

| 角色 | 说明 |
|------|------|
| 访客 | 未登录，可浏览首页公开信息 |
| 学员 | 注册并激活后，使用学员端全部功能 |
| 教练 | 注册并激活后，使用教练端全部功能 |
| 管理员 | 系统内置账号，使用管理员端全部功能 |

### 1.3 通用规则

- **登录方式**：手机号 + 密码
- **注册流程**：学员/教练自助填写信息提交，状态为"待审批"；管理员后台审批通过后账号激活，用户可登录
- **角色路由**：登录后根据角色自动跳转对应端首页（学员→/student，教练→/coach，管理员→/admin）
- **通知机制**：全部站内信，应用内消息中心查看，Tab角标提示未读数
- **机构名称**：Tiger网球俱乐部（管理员可在设置中修改，修改后全局生效）

## 二、认证模块

### 2.1 注册页 `/register`

**访问权限**：访客

**表单字段**：

| 字段 | 类型 | 验证规则 |
|------|------|---------|
| 姓名 | 文本 | 必填，2-10字 |
| 手机号 | 数字 | 必填，11位，唯一性校验 |
| 密码 | 密码框 | 必填，6位以上 |
| 确认密码 | 密码框 | 必填，需与密码一致 |
| 角色 | 单选 | 必选：学员 / 教练 |
| 备注（选填） | 文本 | 最多100字，可填写自我介绍或说明 |

**提交逻辑**：
- 手机号已存在 → 提示"该手机号已注册"
- 提交成功 → 账号状态为 PENDING，跳转至"注册成功"提示页，告知"等待管理员审批后即可登录"
- 管理员审批通过后账号状态变为 ACTIVE，用户方可登录

### 2.2 登录页 `/login`

**访问权限**：访客

**表单字段**：

| 字段 | 类型 | 验证规则 |
|------|------|---------|
| 手机号 | 数字 | 必填 |
| 密码 | 密码框 | 必填 |

**业务规则**：
- 账号不存在 → 提示"手机号或密码错误"（不区分哪个错，防枚举）
- 账号状态为 PENDING → 提示"账号待审批，请等待管理员激活"
- 账号状态为 ACTIVE，密码正确 → 登录成功，跳转对应端首页
- 提供"忘记密码"入口（第一期仅提示"请联系管理员重置密码"）

## 三、学员/家长端

> 底部固定导航栏：首页 | 约课 | 赛事 | 我的

### 3.1 首页 `/student/home`

#### 登录前（访客可见）

**机构Banner区**
- 可滚动轮播图，最多5张，图片由管理员上传配置
- Banner下方显示机构名称（大字）+ 简介（1-2行）

**教练展示区**
- 横向滚动卡片列表，每张卡片包含：教练头像、姓名、一句话简介
- 点击卡片 → 进入教练详情页（头像、姓名、擅长方向、从业年限、个人介绍）
- 只展示状态为"上架/激活"的教练

**课程套餐总览区**
- 图片列表（竖向滚动），由管理员上传套餐介绍图，仅作展示，不可购买（第一期线下购买）
- 只展示状态为"上架"的套餐图片

**底部引导区**
- "登录后可预约课程" 按钮 → 跳转登录页

#### 登录后（学员可见，替换上述内容，保留Banner和教练区）

**已购套餐卡片**
- 显示学员当前有效套餐，每张卡片包含：套餐名称、课时类型（私教/团课）、剩余课时数/总课时数、有效期至
- 无有效套餐时显示"暂无有效套餐，请联系管理员购买"

**待上课课程列表**
- 显示所有"已确认"状态且尚未消课的预约，按上课时间升序排列
- 每条信息：课程类型、教练姓名、上课时间、场地、状态标签
- 最近一节课使用高亮卡片突出显示（距上课时间最近）

**教练月度反馈入口**
- 若有未读月度反馈，显示红点提示
- 点击 → 进入消息通知页，筛选"月度反馈"类型

---

### 3.2 约课 `/student/booking`

顶部两个子Tab：**私教课** | **团课**

#### 私教课子Tab

**步骤一：选择教练**
- 教练卡片列表（头像、姓名、擅长方向）
- 只展示激活状态的教练

**步骤二：选择时段**
- 日历视图，默认展示当天起未来14天
- 每天展示该教练设置的开放时段
- 时段状态：
  - 可预约（绿色/正常）
  - 已被预约（置灰，不可点击）
  - 已过期（置灰，不可点击）
- 选择时段后显示确认信息：教练、日期、时间

**步骤三：提交预约**
- 备注（选填，最多100字）
- 选择消课套餐（下拉选择学员当前有效的私教套餐，显示剩余课时）
- 无有效私教套餐时提示"暂无可用私教课时，请联系管理员"，不可提交
- 提交成功 → 状态变为"待确认"，教练端约课处理模块实时出现该请求，无需站内通知

#### 团课子Tab

**班级列表**
- 每个班级卡片：班级名称、负责教练、上课时间（如"每周三 19:00-20:30"）、适用段位、已报名/总容量、状态标签
- 状态：报名中 / 已满 / 已下架
- 已满班级显示"名额已满"，不可报名

**班级详情页**
- 完整信息：名称、教练、时间、地点（若有）、适用段位、课程介绍、已报名名单（仅显示人数，不显示姓名）
- 课次列表：显示该班级所有课次，状态颜色区分：
  - 已完成：置灰
  - 即将上课：高亮
  - 缺课/请假：橙色标记
- "加入班级"按钮 → 选择消课套餐（下拉选择有效团课套餐）→ 确认加入
- 加入成功 → 状态变为"待确认"，教练端约课处理模块实时出现该请求，无需站内通知

#### 我的预约

**列表视图**（Tab下方入口）
- 筛选Tab：全部 / 待确认 / 已确认 / 已完成 / 已取消
- 每条记录：课程类型（私教/团课）、教练、时间、场地（教练确认后显示）、状态
- "待确认"状态可取消预约（取消后自动退回课时到对应套餐）
- "已完成"状态可查看训练记录（若教练已填写）

---

### 3.3 赛事 `/student/tournament`

**赛事列表页**
- 顶部筛选Tab：全部 / 报名中 / 即将开始 / 已结束
- 赛事卡片：赛事名称、日期、赛制简述、报名截止日、当前报名人数/名额上限、状态标签

**赛事详情页**
- 字段：赛事名称、举办日期、报名截止日期、赛制说明（富文本）、分组说明（按NTRP段位）、名额上限、当前报名人数
- 报名按钮：
  - 已报名 → 显示"已报名"（不可重复报名）
  - 报名中且有名额 → 显示"立即报名"
  - 名额已满 → 显示"名额已满"（不可报名）
  - 已截止/已结束 → 按钮不显示
- 报名确认弹窗：确认参赛信息（姓名、当前段位）→ 确认提交
- 报名成功 → 发站内通知

**我的参赛记录**（"我的"Tab中入口）
- 历届参赛列表，点击展开：赛事名称、参赛日期、所在分组、成绩（管理员录入后显示）、获奖情况
- 若有诊断卡 → 显示"查看诊断卡"按钮，点击查看教练填写内容

---

### 3.4 我的 `/student/profile`

**未登录状态**
- 显示登录/注册引导卡片

**已登录状态**

**个人信息卡**
- 头像（默认头像）、姓名、手机号（部分隐藏：138****1234）
- 点击进入"编辑个人信息"（可修改姓名、密码）

**段位中心**
- 当前NTRP段位（大字展示，如"3.0A"）
- 进度条：当前段位距下一级的进度（基于教练最近一次五维评分）
- 五维雷达图：正手/反手/发球/移动/战术，显示最新评分
- 晋级时间轴：按时间倒序展示每次晋级记录（时间、从哪级升到哪级、审批教练）

**课时包**
- 有效套餐列表（私教/团课分开显示）：套餐名、剩余课时、有效期
- 历史消课记录：按时间倒序，每条记录显示消耗课时、对应课程、操作时间

**消息通知**
- 列表，按时间倒序
- 类型标签：预约通知 / 段位通知 / 赛事通知 / 月度反馈 / 系统通知
- 未读消息标红点，点击后标为已读
- 月度反馈类型：点击展开，显示教练填写的完整反馈内容

**退出登录**

## 四、教练端

> 底部固定导航栏：首页 | 课表 | 学员 | 我的

### 4.1 首页 `/coach/home`

**今日课程汇总卡片**
- 今日共 N 节课，列出每节课简要信息（时间、学员姓名/班级名称）

**待处理约课请求**
- 角标显示待处理数量
- 列表：学员姓名、课程类型、申请时间、期望时段
- 每条操作：
  - **确认**：弹窗填写场地信息（场地名称，选填备注）→ 确认后预约状态变为"已确认"，学员端该预约实时显示场地信息
  - **拒绝**：弹窗填写拒绝原因（必填）→ 确认后预约状态变为"已取消"，自动退回学员课时，发站内通知给学员

**待提交段位评估**
- 角标显示待评估学员数量（管理员指派给该教练的待评估学员）
- 点击进入学员详情页的评估录入入口

---

### 4.2 课表 `/coach/schedule`

**视图切换**：日视图 / 周视图

**日视图**
- 时间轴布局，显示当天所有课程
- 每节课色块：学员姓名（私教）或班级名称（团课）、时间、状态

**周视图**
- 7列布局，每格显示当天课程数量，点击进入当天日视图

**课程详情页**（点击任意课程进入）
- 字段：课程类型、学员/班级、时间、场地、状态、备注
- 若状态为"待确认"：可在此页确认（填场地）或拒绝（填原因）
- 若状态为"已完成"：显示是否已填写训练记录，未填则显示"填写训练记录"按钮

**团课课次出勤核销**（教练端专属操作，课次结束后显示）
- 进入团课课次详情 → 显示本次报名学员列表
- 每位学员行：姓名、当前段位、出勤状态（默认未标记）
- 逐一标记：出勤 / 缺课
- 提交核销 → 出勤学员自动扣除1课时，缺课学员不扣，课次状态变为"已完成"
- 学员端该课次显示对应状态（已完成置灰 / 缺课橙色标记）

---

### 4.3 学员 `/coach/students`

**我的学员列表**
- 显示所有分配给该教练的学员
- 搜索框（按姓名搜索）
- 每行：头像、姓名、当前段位、最近上课日期

**学员详情页** `/coach/students/[id]`

分为四个子Tab：

**基本信息**
- 姓名、手机号（完整显示，教练需联系学员）、注册日期、当前段位

**段位评估**
- 当前五维评分雷达图（只读，展示最新评分）
- "提交新评估"按钮 → 弹窗：
  - 五项评分滑块（各1-5分）：正手稳定性、反手稳定性、发球、移动、战术意识
  - 青少年学员额外显示第六项：比赛心理/竞争意志
  - 评估备注（选填，最多200字）
  - 是否申请晋级（勾选框）→ 勾选后提交给管理员审批
  - 提交成功 → 发站内通知给管理员（若申请晋级）

**训练记录**
- 历史训练记录列表（按时间倒序）
- "添加训练记录"按钮 → 弹窗：
  - 关联课程（下拉选择该学员已完成的课程）
  - 本次训练内容（文本，必填，最多500字）
  - 重点改进项（文本，选填）
  - 提交后学员端"已完成"预约下可查看

**月度反馈**
- 历史反馈列表（按月份倒序）
- "发布本月反馈"按钮（每月只能发一次）→ 弹窗：
  - 本月打得好的地方（文本，必填）
  - 一个值得改进的点（文本，必填）
  - 下阶段训练建议（文本，必填）
  - 发布后：发站内通知给学员，学员端首页反馈入口显示红点

---

### 4.4 我的 `/coach/profile`

- 个人信息（头像、姓名、手机号，可修改姓名/密码）
- 擅长方向、个人简介（可编辑，显示在学员端教练展示区）
- 本月课时统计：已完成课时数（私教/团课分开）
- 薪酬概览：
  - 本月课时费预估（课时数 × 对应提成单价，由管理员配置）
  - 留存奖金进度（本季度负责学员续费率，进度条显示距80%目标的差距）
- 退出登录

## 五、管理员端

> PC端：左侧固定侧边栏导航；手机端：底部Tab（概览/学员/更多）

### 5.1 概览 Dashboard `/admin`

**数据卡片（4个）**
- 学员总数 + 本月新增数（环比箭头）
- 教练总数 + 本月新增数
- 本月已消课时数
- 本季度续费率（有效套餐续费学员数/上季度总学员数）

**待处理事项**
- 待审批账号 N 个 → 点击跳转账号管理
- 待审批晋级申请 N 个 → 点击跳转段位管理
- 进行中赛事报名截止提醒

---

### 5.2 学员管理 `/admin/students`

**列表页**
- 搜索框（姓名/手机号）
- 筛选：按段位（全部/2.5B/2.5A/...）、按状态（全部/激活/待审批/禁用）
- 表格列：姓名、手机号、当前段位、有效套餐、注册日期、状态、操作
- 操作：查看详情、编辑、禁用

**新增学员**（按钮）
- 表单字段：姓名（必填）、手机号（必填，唯一）、初始密码（必填）、备注（选填）
- 提交后账号状态直接为 ACTIVE（管理员直接创建，无需审批）

**学员详情页** `/admin/students/[id]`

分Tab展示：

**基本信息Tab**
- 可编辑：姓名、备注
- 只读：手机号、注册日期、账号状态
- 重置密码按钮

**课时包Tab**
- 当前有效套餐列表：套餐名、类型、剩余课时、总课时、有效期、操作（核销、作废）
- 核销操作：弹窗填写关联课程（选填）、备注（选填）→ 扣除1课时
- 发放套餐按钮 → 弹窗：选择套餐模板（从已上架套餐中选）、有效期起始日、备注（选填）→ 确认后记录线下收款
- 历史核销记录：时间、消耗课时、操作人、备注

**段位Tab**
- 当前段位（大字）
- 历次评估记录（时间、教练、五维评分、是否申请晋级）
- 晋级历史时间轴

**约课记录Tab**
- 全部预约历史（时间倒序），可按状态筛选

**参赛记录Tab**
- 历届参赛信息

---

### 5.3 教练管理 `/admin/coaches`

**列表页**
- 表格列：姓名、手机号、负责学员数、本月课时、状态、操作
- 操作：查看详情、编辑、禁用

**新增教练**（按钮）
- 字段：姓名（必填）、手机号（必填）、初始密码（必填）、擅长方向（选填）、个人简介（选填）
- 提交后账号状态直接为 ACTIVE

**教练详情页** `/admin/coaches/[id]`

- 基本信息（可编辑：姓名、擅长方向、简介；重置密码）
- 负责学员列表（可调整分配关系：给教练新增/移除学员）
- 开放时段配置（见课程管理）
- 本月/本季度课时统计
- 薪酬核算：
  - 课时费单价配置（私教/团课分别设置，元/节）
  - 本月课时费合计
  - 本季度留存奖金计算（奖金规则：续费率≥80%触发，每超1%额外奖励，金额由管理员配置）

---

### 5.4 课程管理 `/admin/courses`

分四个子模块：

#### 套餐管理

**套餐列表**
- 表格列：套餐名称、类型（私教/团课）、总课时数、参考价格、有效期天数、状态（上架/下架）、操作
- 操作：编辑、上架/下架

**新建套餐**（按钮）
- 字段：

#### 团课班级管理

**新建班级**（按钮）
- 基本字段：
  - 班级名称（必填，如"3.0进阶班-周三"或"2025寒假集训班"）
  - 负责教练（必选，下拉选激活教练）
  - 班级类型（必选：长期循环班 / 限期班）
  - 上课地点（选填）
  - 适用段位范围（选填，如"3.0-3.5"）
  - 容量上限（必填，正整数）
  - 单次消耗课时数（必填，默认1）
  - 班级介绍（选填，最多300字）
  - 状态（上架/下架）

- 若选择**长期循环班**，追加字段：
  - 上课星期（必填，如"每周三"）

## 六、通用业务规则

### 课时消耗规则
- 私教课：每节课完成后扣除1课时，未完成（待确认/已确认）不扣
- 团课：每次课完成后扣除1课时（每天单独核销，请假缺课不扣该次课时）
- 取消私教预约（未确认状态）：自动退回课时
- 团课退班：退回剩余未上课时（已上的不退）
- 课时有效期到期后，剩余课时自动清零（系统每日检查）

### 团课课程进度展示规则
每个团课班级包含若干课次，学员报名后可查看该班级的全部课次列表：
- 已完成的课次：置灰显示
- 即将上课的课次：高亮显示
- 缺课/请假的课次：橙色标记

### 团课出勤核销权限
- **主操作方：教练端** — 课程结束后教练逐一标记出勤/缺课，提交后系统自动核销出勤学员课时
- **备用操作方：管理员端** — 可处理教练漏操作、教练离职、学员纠纷等情况
- 同一课次同一学员只能被核销一次，撤销操作仅管理员可执行

### 段位更新规则
- 只有管理员审批通过晋级申请后，学员段位才正式更新
- 教练提交评估但未勾选"申请晋级"时，仅更新五维评分数据，段位不变
- 同一学员不可同时存在两个待审批的晋级申请

### 通知触发规则

预约相关的交互（提交、确认、拒绝、取消）通过**预约状态变更**实时呈现，不发站内通知。其余事件触发站内通知如下：

| 触发事件 | 通知接收方 |
|---------|-----------|
| 教练拒绝预约 | 学员（告知拒绝原因） |
| 教练提交晋级申请 | 管理员 |
| 管理员审批晋级通过 | 学员 + 教练 |
| 管理员驳回晋级申请 | 教练 |
| 教练发布月度反馈 | 学员 |
| 赛事报名成功 | 学员 |
| 诊断卡发送 | 学员 |
| 新账号注册 | 管理员 |
| 管理员激活账号 | 学员/教练 |
